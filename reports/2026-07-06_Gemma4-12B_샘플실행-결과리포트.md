## 1. 샘플이 무엇인가

**다시,출근** = 40~50대 중장년 재취업 지원 서비스. Gemma 4 12B 하나로 6개 텍스트 태스크를 처리한다(태스크별 system 프롬프트 + 생성 파라미터만 다름). 결정론/규칙 엔진이 낸 값(점수·가능시간·역량분석)을 **모델이 "언어화"**하는 구조 — 모델이 숫자를 만들지 않고 설명만 한다는 게 설계 핵심.

입력은 목업(자동차 부품 QC 25년 경력 구직자). 아래 출력은 전부 **모델이 실제로 생성한 원문**이다.

## 2. 태스크별 실행 결과 + 평가

### ① 온보딩 추출 — 상담 대화 → 구조화 JSON
입력: 상담 대화 4줄(경력·자격·희망지역 언급, 이름·희망직종은 미언급).
```json
{
  "이름": null,
  "총_경력_년수": 25,
  "주요_직무": ["자동차 부품 품질검사"],
  "보유_기술_자격": ["지게차 면허", "측정기 조작"],
  "최근_직장_직위": "반장",
  "희망_직종": [],
  "근무_가능_지역": ["경기 남부"],
  "제약_사항": null
}
```
**평가:** 스키마 8개 필드 정확. **대화에 없는 이름·희망직종을 지어내지 않고** null/[]로 처리(system 규칙 "추측 금지" 준수). 25년·반장·지게차·측정기·경기남부 모두 대화에서 정확히 추출.
**주의:** 추출형은 모델이 가끔 코드펜스/설명을 붙여 JSON 파싱을 깨뜨릴 수 있음 → `client.onboarding_extract`가 펜스 제거 후 파싱, 실패 시 `_raw` 반환. 운영에선 **재시도 1회** 추가 권장.

### ② 자소서 초안
입력: 공고(품질관리 담당) + 경력(QC 25년, 반장 8명, 지게차, 3차원 측정기).
```
25년간 자동차 부품 제조 현장에서 품질검사 업무를 수행하며 정밀한 품질 관리 역량을
쌓아왔습니다. 단순히 제품을 검수하는 것에 그치지 않고, 공정 전반의 흐름 속에서
발생할 수 있는 불량 요인을 사전에 파악하고 대응하는 데 집중했습니다.

특히 3차원 측정기 등 전문 장비를 능숙하게 운용하며 정밀한 치수 관리와 품질 기준
준수를 실현해 왔습니다. 또한, 현장에서 8명의 팀원을 이끄는 반장 역할을 수행하며
협업을 통한 공정 효율화와 후배 양성을 경험했습니다.

이러한 숙련된 장비 활용 능력과 현장 관리 경험은 귀사의 품질관리 직무에서 즉각적인
성과로 이어질 것입니다. 원칙에 기반한 꼼꼼한 검수와 안정적인 팀 운영으로 제품의
신뢰도를 높이는 데 기여하겠습니다.
```
**평가:** **경력 사실만 사용**(없는 수치·자격 창작 없음). 공고 우대사항(측정장비·품질검사)에 맞춰 경력을 앞세워 재배치. 중장년 강점(책임감·후배 양성)을 나이 언급 없이 자연스럽게 녹임. 미사여구 절제, 문단형 500자 내외 — system 규칙 5개 모두 충족. **가장 강한 태스크.**

### ③ 꼬리질문 Q4~Q5
입력: 지원자 Q1~Q3 답변(지원동기·강점·팀관리).
```
Q4. 데이터 분석을 통해 불량 원인을 파악하고 재발 방지 대책을 세우셨던 구체적인
    사례와 당시 활용하신 핵심 지표를 말씀해 주실 수 있습니까?
Q5. 8명의 팀원을 이끄는 반장으로서, 교대 근무 중 발생할 수 있는 현장의 갈등이나
    일정 조율의 어려움을 어떻게 해결하셨는지 구체적인 경험을 들려주십시오.
```
**평가:** 지원자가 실제 말한 내용("데이터로 불량 원인", "반장 8명 교대")에만 근거해 **한 단계 더 구체화**를 요구. 압박·함정이 아니라 역량을 드러낼 기회형. 존댓말·2~3문장 규칙 준수.

### ④ 면접 피드백
입력: 답변 요약 + 역량 분석(분류기 산출: 직무 상/전달력 중/리더십 정보부족).
```
현장에서 쌓아오신 풍부한 경험이 답변 속에 잘 녹아있습니다. ...
다만, 그 성과가 얼마나 컸는지 ... '수치'를 활용한 표현이 보완되면 ...
"불량률을 몇 % 낮췄다"거나 "공정 시간을 얼마나 단축했다"와 같은 ...
다음 면접에서는 ... 1. 성과 수치화 ... 2. 리더십 에피소드 추가 ...
풍부한 실무 경험은 가장 큰 무기입니다. ... 함께 차근차근 준비해 나갑시다.
```
(전문은 부록 파일 참조)
**평가:** **합격/불합격·점수를 말하지 않음**(system 규칙 준수). "잘 전달된 점 → 보완점 → 다음 행동" 순서 정확. 역량 분석의 약점(전달력 중=수치 부족, 리더십 정보 부족)을 **구체적 행동 제안으로 변환**. 중장년 자존감 배려 톤, 빈 칭찬 없음.

### ⑤ 매칭 근거
입력: 규칙 산출 점수(종합82 / 직무92 / 지역100 / 근무형태60).
```
선생님께서 쌓아오신 25년의 품질검사 경력이 해당 직무와 매우 잘 맞아떨어져 높은
점수로 추천해 드립니다. 특히 거주하시는 경기남부 지역 내에 위치하여 출퇴근이
편리하다는 점도 큰 장점입니다. 교대 근무가 포함되어 있으나, 숙련된 기술을 발휘하며
안정적으로 근무하실 수 있는 자리입니다.
```
**평가:** 입력 점수를 **바꾸거나 새 숫자를 지어내지 않고** 근거(직무 일치·지역·근무형태)만 일상어로 풀이. 낮은 항목(근무형태 60=교대)도 회피 않고 담담히 언급. 2~3문장 규칙 준수.

### ⑥ 일정 제안
입력: 확정 가능 시간대(6/12 목 14~16시, 6/13 금 10~11시).
```
면접 일정과 관련하여 안내드립니다. 확인 결과, 6월 12일(목) 14시~16시 사이 또는
6월 13일(금) 10시~11시 사이에 면접 진행이 가능합니다. 위 시간대 중 편하신 시간을
말씀해 주시면 확정하도록 하겠습니다. 혹시 해당 시간에 조율이 어려우실 경우, 편하게
말씀해 주시면 추가로 일정을 확인해 드리겠습니다.
```
**평가:** **주어진 시간대 밖을 제안하지 않음**(범위 준수). 정중한 존댓말 + 대안 안내 포함. 3~4문장 규칙 준수.

## 3. 종합 평가 — Gemma 4 12B는 이 서비스에 쓸 만한가

**결론: 예. 무료 티어(로컬 Q4) 후보로 충분하다.**

- **강점:** 한국어가 매끄럽고, 특히 **"규칙을 지키는" 순종성**이 좋다 — 점수 창작 금지, 판정 금지, 범위 밖 제안 금지, 없는 사실 창작 금지를 6개 태스크에서 모두 지켰다. 중장년 대상 서비스의 **톤(담백·존중)** 재현이 안정적.
- **약점/주의:**
  1. **JSON 추출 형식 안정성** — 생성형은 완벽하나 구조화 출력은 가끔 깨질 수 있어 파서+재시도 방어가 필수(코드에 일부 반영됨).
  2. **자소서 few-shot이 현재 1개** — 3~5개로 늘리면 중장년 톤이 더 고정된다(파인튜닝보다 먼저 할 일).
  3. **속도** — 로컬 CPU/iGPU 기준 느림(체감 응답 수 초~십수 초). 상시 데모는 GPU 서빙 권장.
- **다음 단계 제안:** ① 자소서 few-shot 확장 ② 온보딩 재시도 로직 ③ `config.MODEL` 한 줄만 바꿔 26B/Qwen과 **blind bake-off**(자연스러움/구체성/중장년 톤 + "12B의 손실률") 후 무료 티어 확정.

## 4. 현재 RAG 구성 (how it's built)

샘플의 `client.py`에는 RAG "seam"(주입 지점)만 주석으로 있었고, 실제 검색 구현이 없었다. 이번에 **`rag_demo.py`로 그 seam을 실제 구현**했다. 현재 구성은 다음과 같다.

### 4-1. 아키텍처
```
[문서(공고 JD 등)]                     [지원자 경력 = 쿼리]
        │                                     │
        ▼  bge-m3 (Ollama /v1/embeddings)     ▼  bge-m3
   문서 벡터[N×1024]                      쿼리 벡터[1024]
        └───────────────┬─────────────────────┘
                        ▼  코사인 유사도 (numpy)
                   top-k 공고 선택 (k=2)
                        ▼
              최상위 공고 + 지원자 경력
                        ▼  client.resume_draft() → gemma4-12b-text
                   맞춤 자기소개서
```

### 4-2. 구성 요소

| 요소 | 현재 구현 | 비고 |
|---|---|---|
| 임베딩 모델 | **bge-m3** (Ollama, `ollama pull bge-m3`) | 1024차원, 다국어(100+), 8192토큰 |
| 임베딩 호출 | OpenAI 호환 `/v1/embeddings` | `_emb.embeddings.create(model="bge-m3", input=...)` |
| 벡터 저장/검색 | **numpy 코사인 유사도** (인메모리) | 소규모 데모용. 정규화 후 내적, `argsort`로 top-k |
| 지식베이스 | 코드 내 `JOB_DOCS`(공고 4개) | 데모 고정 데이터 |
| 생성 | `gemma4-12b-text` (`client.resume_draft`) | 검색 결과를 프롬프트에 주입 |
| 주입 방식 | 검색된 공고 텍스트를 `job_posting` 인자로 전달 | `prompts.build_resume_user`가 `[채용공고]` 블록에 삽입 |

### 4-3. 실제 동작 결과
```
── 지원자 경력에 가장 맞는 공고 검색(top-2) ──
   1. sim=0.671  품질관리 담당(제조). 우대: 품질검사 경력, 측정장비 운용, 불량 원인 분석.
   2. sim=0.600  생산라인 반장. 우대: 라인 운영, 작업자 교육, 생산 실적 관리.
```
QC 25년 경력 쿼리 → 4개 공고 중 "품질관리 담당(제조)"을 top-1로 정확히 선택 → 그 공고로 자소서 생성.
(생성 자소서 전문: [outputs/rag_demo_output.txt](outputs/rag_demo_output.txt))

### 4-4. 현재 한계 & 실서비스로 갈 때 바꿀 점

| 현재(데모) | 실서비스 권장 |
|---|---|
| numpy 인메모리 검색 | **FAISS**(또는 pgvector) 인덱스 — 문서 수 커질 때 필수 |
| 지식베이스 하드코딩(4건) | 실제 공고 DB에서 적재 + 주기적 임베딩 갱신 |
| 검색=공고만 | 공고 + **합격 자소서 예시** 동시 검색해 few-shot로 주입(README 4·6항) |
| dense(코사인)만 | bge-m3의 **sparse/multi-vector** 혼합 + **bge-reranker-v2** 재랭킹 |
| 청킹 없음 | 긴 문서는 청킹 후 임베딩 |

**요지:** 현재 RAG는 "bge-m3 임베딩 → 코사인 top-k → 프롬프트 주입 → gemma 생성"의 **최소 동작 파이프라인**이 검증된 상태다. 자소서 품질은 모델 체급보다 **주입되는 공고 JD·경력 조각·합격 예시의 질**이 더 좌우하므로, 다음 투자는 모델 교체보다 **검색 품질(reranker)·few-shot 예시 확장**에 두는 것이 효율적이다.
