## 1. 할루시네이션 방지 (Anti-hallucination)

- 추측으로 단정하지 않는다. 파일·심볼·경로·설정값·API 응답 형태는 **실제 저장소/코드를 읽어 확인한 뒤** 언급한다.
- 존재하지 않는 함수/파일/플래그/엔드포인트/라이브러리를 **지어내지 않는다.** 불확실하면 "확인 필요"라고 명시한다.
- 코드·수치·경로를 인용할 때는 실제 파일 위치(`path:line`)로 근거를 댄다.
- 외부 라이브러리/모델/가격/한도/공공 API(고용24·data.go.kr·Google Calendar 등)는 **기억에 의존하지 말고** 공식 소스나 코드로 검증한다. 검증 시점(날짜)을 함께 남긴다.
- 검증하지 못한 내용은 "검증 못 함 / 미확인"으로 분명히 표시하고, **사실처럼 보고하지 않는다.**
- 데모용 시드 데이터·목업 수치는 코드/문서에 **"예시 데이터(seed)"라고 명시**한다. 실측·실데이터인 것처럼 제시하지 않는다.
- 완료·검증을 보고할 때는 실제 실행 결과(임포트·부팅·엔드포인트 응답·테스트 출력)를 근거로만 "된다"고 말한다. 실패는 실패라고, 건너뛴 건 건너뛰었다고 **가감 없이** 보고한다.

## 2. AI 하네스 기법 적용 (Apply AI harness techniques)

- 다단계·복합 작업은 **계획 → 추적(Task 리스트) → 실행 → 검증** 순서로 진행한다.
- 시작 전 `plans/{작업분야}-{작업명}.md` 형식의 계획서를 먼저 작성한다(목표 / 영향 파일 / 단계 / 위험·롤백 / 검증 방법). `plans/` 는 로컬 참고용.
- 독립적인 조사/검색은 서브에이전트로 병렬화하고, **결론만 회수**한다(파일 덤프를 컨텍스트에 끌어오지 않는다).
- 구조화가 필요한 산출물은 **스키마 기반 출력**으로 받는다.
- 되돌리기 어렵거나 파괴적인 작업(폴더 비우기 / 덮어쓰기 / 삭제 / 외부 전송 / 푸시)은 **실행 전 확인**한다. 대상이 설명과 다르면 진행하지 말고 **먼저 보고**한다.
- 변경 후에는 실제로 동작하는지(임포트·부팅·엔드포인트·테스트)를 확인하고 결과를 보고한다.
- GUI/스타일 수정은 **요청 범위만 최소 변경(minimal diff)**. 요청하지 않은 레이아웃·색·문구·구조는 건드리지 않는다.

## 3. 하드코딩·폴백 최소화 (Hardcoding & fallback only where necessary)

원칙: **하드코딩과 폴백은 "꼭 필요한 곳"에만.** 편의를 위해 습관적으로 심지 않는다.

### 3.1 허용되는(=필요한) 경우 — 반드시 라벨링
- **외부 의존성 부재 시 데모 지속용 폴백**
  - LLM API 키 미설정 → 템플릿 기반 생성으로 폴백. (`infrastructure/llm/client.py`)
  - 공공 API(고용24·data.go.kr) 미승인/오프라인 → 로컬 **시드 데이터셋**으로 폴백. (`infrastructure/data/*`)
  - 화상 API(구루미 등) 미확보 → Google Meet 형식 링크로 폴백.
  - DB 미설정 → 인메모리 저장소 폴백. (GAJIMA 의 "memory 폴백" 패턴 계승)
- 위 각 폴백은 코드 주석과 응답 `meta` 에 **`source: "seed"|"fallback"|"live"`** 로 출처를 드러낸다.

### 3.2 금지되는(=불필요한) 경우
- 계산으로 도출 가능한 값을 상수로 박아두기(예: 적합도·교집합 시간은 **로직으로 계산**, 미리 정답을 심지 않는다).
- 에러를 숨기려는 광범위 `try/except: pass` 또는 조용한 기본값 반환. (실패는 드러낸다)
- 데모를 "그럴듯하게" 보이려는 가짜 결과 하드코딩.

### 3.3 결정론 로직 vs LLM 언어 계층 (설계 분리)
- **일정 확정·매칭 점수·역량 갤 산정**은 결정론적 로직(순수 함수, `domain/`). LLM 에 맡기지 않는다.
- **LLM 은 언어 계층에만**: 대화형 정보 수집, 서류 초안 문장 생성, 자연어 제약 해석, 대안 제안문 작성.
- 이 경계를 흐리지 않는다. (팀 설계 원칙 — `init_plans/중장년_재취업_실행에이전트_팀공유브리핑.html` §04 근거)

## 4. 응답 계약 (API response envelope)
- 모든 HTTP 응답은 `{ok, data, meta, error}` 봉투를 따른다. (GAJIMA `interfaces/http/responses.py` 패턴 계승)
- `meta.source` 로 `live`/`seed`/`fallback` 을 항상 표기한다.

## 5. 이력 관리 (History)
- 의미 있는 변경(구조 셋업, 기능 추가, 버그 수정, 릴리스)은 `HISTORY.md` 에 날짜와 함께 **한 줄 이상** 기록한다.
- 버그·연동오류 해결 시 `reports/BUG-{NNN}_{슬러그}.md` 를 작성한다(증상·위치 `path:line`·원인·해결·검증). 추측 금지(§1).
