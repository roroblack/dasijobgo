## 3. RAG — 두 갈래로 검증 (주입 지점이 다름, 상보적)

첨부 모듈의 `resume_draft`는 두 RAG 주입점을 가짐 — **`job_posting`(어떤 공고냐)** 과 **`rag_examples`(유사 합격 예시 톤)**. 두 환경이 각각 하나씩 검증:

| | (B) 12B 팀 — `rag_demo.py` | (A) 본 세션 — `run_rag_bakeoff.py` |
|---|---|---|
| 임베딩 | **bge-m3** (Ollama /v1/embeddings, 1024d) | ko-sroberta(110M) — **BGE-M3는 노트북 RAM 초과(OOM) 대체** |
| 주입점 | **공고 검색** → `job_posting` | **유사 합격예시 검색** → `rag_examples`(few-shot) |
| 검색 | 코퍼스 공고 4건, 코사인 top-2 | 코퍼스 예시 5건, 코사인 top-2 |
| 결과 | QC 쿼리 → "품질관리 담당" **top1 sim 0.671** | QC 쿼리 → 품질 예시 **top1 cos 0.769** |
| 판정 | 검색·주입 파이프라인 정상 동작 | 검색 정상, 단 **약한 2순위(0.535) 주입이 새 오염** |

**통합 결론**
- **검색 자체는 두 환경 모두 정상**(동일 직무를 top1으로).
- **주입 설계가 관건**: 본 세션에서 top-k 무차별 주입(약한 2순위 포함)이 "물류/무재해 3년" 새 오염을 유발 → **유사도 임계값(≥0.65)·top-1·"톤 전용" 주입**이 필요.
- 12B 팀도 동일하게 **다음 투자 = FAISS/pgvector·reranker(bge-reranker-v2)·합격예시 few-shot·청킹**을 지목. → **RAG 품질은 모델 체급이 아니라 검색·주입 설계에 달림**(양측 일치).

## 4. ★ 교차검증 — 두 독립 평가의 수렴

약한 노트북(소형 4bit)과 강한 머신(12B)에서 **독립적으로 같은 결론**에 도달:

| 발견 | (A) 소형 모델 실측 | (B) 12B 실측(첨부) |
|---|---|---|
| few-shot 1개는 부족 | Kanana가 물류센터 예시 베낌 | "few-shot 3~5개로 확충, 파인튜닝보다 먼저" |
| JSON 추출 방어 필요 | EXAONE 비스키마 키·0.5B 깨짐 | "형식 가끔 깨짐 → retry 로직" |
| RAG 다음 투자 = 검색/예시 | 임계값·top-1·톤전용 필요 | "reranker·합격예시 few-shot" |
| 모델 체급 < 설계 | 2B로도 정상, 병목은 주입 | "품질은 모델 체급보다 JD·경력·예시 질" |

→ **결론: 모델 갈아끼우기보다 (1) 자소서 few-shot 확충 (2) RAG 임계값/reranker (3) JSON retry 가 우선 투자.** 두 관측이 이를 강하게 뒷받침.

## 4-B. ★ 설계 레버 실증 — 가이드 정렬 전/후 재측정 (2026-07-06 추가 실행)

§4에서 예측한 "few-shot 정렬" 레버를 **실제로 적용·측정**. `prompts.py`의 자소서 프롬프트를 고용24 가이드에 맞춰 정렬:
- 원본 물류센터 **단일 예시** → 마스킹(○○) **다중 예시**(생산관리 + 품질관리)
- system에 **"예시의 회사·업종·수치를 지원자 것으로 가져오지 말 것"** 규칙 추가(+ STAR·지원동기 관점)

동일 모델·동일 입력(품질관리 지원자)으로 OLD vs NEW 자소서 생성, 오염 지표 비교(`output/fewshot_align_compare.md`):

| 모델 | OLD(정렬 전) 오염 | NEW(정렬 후) 오염 | 25년(본인) 유지 |
|---|---|---|---|
| EXAONE-2.4B | 물류센터 ×3 | **없음** | OLD·NEW 모두 ✅ |
| Kanana-2.1B | 물류 ×2 + **22년(오년)** | 무재해 ×1(약함) | OLD ✗ → **NEW ✅ (25년 교정)** |

- **EXAONE**: 물류센터 오염 3 → **0**(완전 제거). 단 NEW가 "자소서:…" 머리말+굵은글씨를 붙임 → "본문만" 규칙 경미 위반(규칙 강화 여지).
- **Kanana**: 앞서 최악 사례(**22년 오년 + 물류**)가 **25년 정확 + 물류 제거**로 교정. "무재해" 1회 약한 잔류만 남음.
- **결론: 정렬(설계 레버)이 오염을 실제로 줄임.** 모델 교체 없이 프롬프트만 고쳐서, §4 두 평가가 예측한 개선이 **수치로 확인**됨 — RAG·파인튜닝보다 먼저 할 값싼 개선.
- ⚠️ 1회 표본(temp 0.6)·완전 제거는 아님(Kanana 무재해 잔류, EXAONE 머리말). 예시 3~5개 추가 확충 + 규칙 강화로 추가 개선 여지.

## 6. 한계·주의 (RULE §1 — 실측/추정 구분)
- **경향치이지 확정 순위 아님**: 각 모델 1회·temperature>0·mock 1건. 확정은 **동일 프롬프트 10건 × blind 채점** 필요.
