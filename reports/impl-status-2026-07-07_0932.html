설계(3대 축 · 화면 12단계 · 기능 계층) 대비 실제 코드 상태. 이번 배치로 역량 근거추출(NCS)·STT·TTS·마이크 배선·웹캠 녹화가 구현됨(앞선 LLM 로컬+RAG 포함). 진행률 %는 코드 기준 추정.

✓이번 배치 — 미구현 → 구현 (실동작 검증)

  • ① 역량 근거추출 (NCS rubric) — domain/rubric.py: 답변 텍스트 → 5축 지표 밀도 점수 + 근거 문장, 정량화 축은 숫자 유무 판정 → /interview/analyze-answer. pytest 11/11. 결정론(RULE §3.3).
  • ② STT 음성 입력 (서버측 실모델) — /stt = faster-whisper(CPU·int8). 검증: 한국어 음성(webm/opus) → 전사 "…품질 관리를 25년 했습니다". 전사는 프론트가 아니라 서버에서 수행.
  • ③ TTS 아바타 발화 — STEP 8 아바타가 질문을 브라우저 speechSynthesis(한국어)로 읽음(검증: 실제 발화).
  • ④ 마이크 배선 — STEP 1 마이크 = getUserMedia+MediaRecorder 녹음 → /stt 전사 → 칩 자동 매칭. 짧음/무음 가드.
  • ⑤ 웹캠 녹화 — STEP 7 실제 카메라 미리보기·장치상태, STEP 8 라이브 셀프뷰 + MediaRecorder 답변 녹화. 장치 없으면 폴백.
  • (앞서) LLM 문장 생성 로컬 GGUF(Gemma-4-E4B, source=local) · RAG 임베딩 검색(유사도 임계값).
STT 아키텍처: 프론트는 녹음→업로드만, 전사는 서버 faster-whisper. "(안 들렸어요)"는 그 녹음이 짧/무음일 때의 폴백 — webm/opus 서버 전사는 정상 검증됨(디코딩 버그 아님).
⚠️ 전부 env-gated(모델·장치 없으면 폴백, 데모 지속). 프리뷰(무장치)에선 폴백 경로만 자동검증 — 실제 마이크/카메라 캡처는 실기기 필요. 신경망 TTS·아바타 립싱크는 GPU 필요로 잔존.
