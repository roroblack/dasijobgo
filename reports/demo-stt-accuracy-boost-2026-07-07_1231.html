3. 추천 — 데모용 우선순위

1

리턴제로(RTZR) STT API 연동 — 가장 큰 즉효

정확도 2배(자체엔진 CER 5.91%, 상담셋 3.51% — 중장년/전화 유사 도메인 최강). 우리에게 딱 맞는 3가지가 이미 내장:
  • 간투어("음·어·그") 자동 제거 → 우리 슬롯추출 입력이 깨끗해짐(하이브리드 규칙층과 정확히 시너지)
  • 키워드 부스팅 → 품질관리·지게차·설비보전 등 직무어휘 인식률↑ (커스텀 사전)
  • 실시간 스트리밍(<300ms) → 이전에 논의한 실시간 UX 그대로 가능
무료 10시간(가입 즉시)로 데모/발표 전부 커버. 국내 서버라 개인정보 측면 미국 API보다 유리(온프레미스 옵션까지 있음). CPU/RAM 부담 0(서버가 대신 처리) → 우리 랩탑 자원 해방.
현재 large-v3 오류율(≈11%) ▔▔▔
RTZR 오류율(≈5.9%) — 막대 짧을수록 좋음
구현: STT 클라이언트에 RTZR 백엔드 추가(HTTP, env-gated) → 우선순위 RTZR → 로컬 whisper 폴백. 기존 meta.source 패턴에 자연히 얹힘. 약 1파일 + API 키.
유의: 네트워크 의존 + 오디오가 RTZR 클라우드로 전송됨 → 데모라도 이용자 동의 문구 권장(온프렘 쓰면 egress 자체가 사라짐).
2

한국어 파인튜닝 Whisper 드롭인 (로컬·무비용·오프라인)

남이 한국어로 파인튜닝해 공개한 모델을 우리가 학습 없이 교체. 온디바이스라 PII 유출 0·비용 0·네트워크 불필요 — 정부지원 서비스에 가장 안전.

정확도가 오르는 원리 — 3층이 쌓임:
  • ① 한국어 전용 가중치(핵심): 바닐라 large-v3는 99개 언어 다국어라 한국어 용량이 일부 → 받침·유사음 혼동, 띄어쓰기 붕괴, 한국어 환각. 한국어 추가학습 모델은 음소→한글 매핑·한국어 LM prior가 강화돼 이 오류가 줄어듦(ENERZAI: 한국어 토크나이저+철자정규화로 CER 절반).
  • ② 도메인 어휘 주입(우리 몫): hotwords/initial_prompt로 품질관리·지게차·설비보전 등 직무어휘 편향 — 학습 0(RTZR 키워드부스팅의 로컬판).
  • ③ 무료 디코딩 레버: turbo 금지·VAD 완화·저신뢰 세그먼트 필터(기적용)가 ①② 위에 순증.
B에서 가장 큰 레버 = "어떤 파인튜닝을 고르냐". 학습 데이터가 낭독체(zeroth·CER 2.06% 헤드라인)면 깨끗한 낭독엔 최강이나 즉흥·중장년엔 그만큼 안 나옴. 자유발화(KsponSpeech 계열)로 학습한 모델을 골라야 우리 케이스에 전이가 좋다 → 후보 2~3개를 우리 오디오로 A/B 선정.

정직한 천장: 현실적 ~11% → ~7~8%(자유발화 파인튜닝 잘 고를 때). 헤드라인 2%도 아니고 RTZR 5.9%를 꼭 이긴다는 보장 없음. 지연(~15s)은 그대로(단 turbo-korean 고르면 더 빠름). → 55+ 실측으로 확정, 아니면 폴백.
구현 순서: ①후보 A/B(eval_cer.py) → ②ct2-transformers-converter --quantization int8(finetune/to_ctranslate2.py) → ③STT_MODEL=<ct2경로> 재시작(백엔드 무수정) → ④hotwords 주입 → ⑤55+ 실측 확정.
3

무료 설정 레버 — 무엇을 고르든 항상 병행

  • hotwords: faster-whisper hotwords="품질관리 지게차 설비보전 물류 생산관리 인천 남동구…" — 학습 0으로 도메인어휘 부스팅(RTZR 키워드부스팅의 로컬판). 즉시
  • VAD 완화: 중장년 느린 발화가 문장 중간 침묵에 잘리지 않게 min_silence↑.
  • large-v3 유지(turbo 금지): 반복환각 최소 — 이미 적용됨.
  • initial_prompt(한국어 상담 도메인) — 이미 적용됨.
효과는 크지 않지만 비용 0·즉시. A나 B 위에 얹으면 순증.

4. 개인정보(PII) 관점 — 놓치면 안 됨

「다시,출근」은 정부지원·중장년 개인 경력정보를 다루는 서비스다. 음성엔 이름·경력·거주지가 담긴다. 따라서 전사 위치가 중요:
  • 가장 안전: B(온디바이스 로컬) — 오디오가 기기를 안 떠남.
  • 안전: A(리턴제로) — 국내 서버, 온프레미스 선택 시 사실상 로컬. CLOVA도 국내.
  • 주의: E·F(OpenAI/ElevenLabs/Deepgram) — 미국 서버 전송 → 국내 개인정보 처리 관점에서 데모라도 신중, 실서비스면 재검토.
→ 정확도 최상 + 국내 데이터 잔류를 동시에 만족하는 리턴제로(온프렘 옵션 포함)가 이 서비스 성격에 가장 부합.

5. 권고 실행안 (데모 기준)

바로 (오늘~내일)

  • RTZR 가입 → 무료 10h 키 발급(사용자님)
  • STT 클라이언트에 RTZR 백엔드 추가, RTZR→로컬 폴백 env-gate (나)
  • 키워드부스팅에 직무사전 주입 + 로컬엔 hotwords 병행 (나)
  • 이용자 음성 동의 문구 1줄 (나)

대안 (오프라인·무비용 필수 시)

  • ghost613 한국어 Whisper 다운 → CT2 변환 → STT_MODEL 교체
  • hotwords + VAD 완화 병행
  • 확보 예정 55+ 데이터로 RTZR vs 로컬 CER 실측 비교(finetune/eval_cer.py 재사용)
제 최종 추천: 데모는 A(리턴제로) + C(무료레버). 이유 = 오늘 정확도를 2배로 올리는 유일한 길이면서, 간투어제거·키워드부스팅·스트리밍이 우리가 이미 만든 슬롯추출/실시간UX와 정확히 맞물리고, 국내 서버라 개인정보에도 부합. 오프라인이 절대조건이면 B로. 파인튜닝(별도 트랙)은 그 후 "8%→X%" 측정으로 정당화되면 진행.
